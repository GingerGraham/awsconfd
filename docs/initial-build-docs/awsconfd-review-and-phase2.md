# `awsconfd` — Review Findings and Phase 2 Plan

This document has two parts.

**Part A** is a remediation list for the code as delivered. It must be worked before Part B, because Phase 2 depends on primitives that are currently broken (`_parse_sections`, staleness detection, global flag placement).

**Part B** specifies the `add-sso`, `add-profile`, `apply --spec`, `watch` and `hook` commands.

Everything in Part A was reproduced by running the delivered script, not inferred by reading it.

---

# PART A — Remediation

## A0. The `set -e` class of bug

This is the root cause of most of the silent failures. Fix it first and globally.

### A0.1 `((var++))` returns exit status 1 when `var` was 0

```bash
set -e; n=0; ((n++))    # exits here
```

An arithmetic command's exit status is 1 when the expression evaluates to zero. `((n++))` evaluates to the *pre-increment* value. Under `set -e` this terminates the shell.

Present at: `_parse_sections` (`((line_no++))` — dies on the first line of every file), `cmd_init` (`((backup_num++))`), `_validate_no_dups_across`, `_validate_all`, `cmd_doctor`.

**Consequence, verified:** `_parse_sections` never returns a single section. `awsconfd list` exits 1 with no output. `doctor` runs no validation. Duplicate detection never fires. All of it silent.

**Fix:** `n=$(( n + 1 ))` everywhere. Never `(( ))` as a standalone command under `set -e` unless suffixed `|| true`. Add a grep test (§A9).

### A0.2 A function whose last statement is `[[ cond ]] && cmd` returns 1 when `cond` is false

```bash
_info() { [[ $QUIET -eq 0 ]] && echo "$*" >&2; }   # returns 1 under --quiet
```

The `&&` list's status is 1. `set -e` ignores that *inside* the list, but the function returns 1 and the **caller's** invocation of `_info` is then a failing command, which does trigger `set -e`.

Present at: `_info`, `_success`, and every `[[ … ]] && _info …` used as a final statement.

**Fix:** every logging function ends with an explicit `return 0`:

```bash
_info() {
    if [[ $QUIET -eq 0 ]]; then
        printf '%sINFO:%s %s\n' "$BLUE" "$RESET" "$*" >&2
    fi
    return 0
}
```

Same for `_warn`, `_err`, `_success`. `_err` should still `return 0` — its callers decide the exit code.

### A0.3 Audit

After fixing, add to the test suite a run of every command under `-q`, under `-v`, and with an empty `config.d`, asserting the expected exit code. `set -e` failures are invisible without this.

## A1. Values assigned inside `( … )` subshells are lost

### A1.1 `cmd_status` — the worst instance

```bash
(
    shopt -s nullglob
    for f in "$AWSCONFD_CONFIG_DIR"/*.conf; do
        ...
        stale=1        # lost
    done
)
```

**Verified:** `touch ~/.aws/config.d/20-a.conf; awsconfd status --check` → exit 0. Fragment edits are not detected as stale.

This function is the correctness backstop for the entire watcher design (Layers 2, 3 and 4 all depend on it). It must be right.

### A1.2 `_validate_no_dups_across`

```bash
... | while read -r dup; do
    _err "..."
    ((errors++))       # subshell
done
return $errors         # always 0
```

Duplicate sections are printed but never block. `_validate_all`'s `|| ((errors++))` therefore never fires.

**Fix pattern for both:** avoid pipelines and subshells where a result must escape. Use a temp file, or process substitution is unavailable in the portability budget, so:

```bash
_config_stale() {
    local d="$AWSCONFD_CONFIG_DIR" c="$AWSCONFD_CONFIG_FILE"
    [[ -d $d ]] || return 1
    [[ -f $c ]] && return 0        # no config → stale
    [[ $d -nt $c ]] && return 0    # directory mtime: catches create/delete/rename
    local f
    shopt -s nullglob
    for f in "$d"/*.conf; do
        [[ $f -nt $c ]] && return 0
    done
    return 1
}
```

`[[ a -nt b ]]` is a bash builtin. **No `stat`, no fork.** Note `shopt -s nullglob` must be set in the main shell (set it once near the top, and set `shopt -u nofile`-style options nowhere else), not inside a subshell.

## A2. `stat -c` / `stat -f` are used and are forbidden

Lines 482, 485, 494 of the delivered script. The spec forbids them (GNU vs BSD incompatibility). The `stat -c … || stat -f …` fallback also leaves `$config_mtime` empty when the file is absent, and `[[ "" -gt 5 ]]` is an arithmetic error.

Replace entirely with `-nt` as above.

## A3. Data-loss defects

### A3.1 `build` overwrites a hand-written config with no backup

**Verified:** with a hand-written `~/.aws/config` and an empty `config.d`, `awsconfd build` replaces it with a banner-only file. The original is gone.

**Fix — implement spec §6.5 in `build`:**

Before installing the new file, if the existing output file exists **and** does not contain the generated banner's sentinel line, take a numbered backup first. Sentinel: an exact-match grep for `# DO NOT EDIT THIS FILE BY HAND.` (or better, a dedicated machine-readable line — see A5).

This is not optional. It is the difference between a tool and a footgun.

### A3.2 `init` is not idempotent, and overwrites `00-defaults.conf`

```bash
if [[ -d $AWSCONFD_CONFIG_DIR ]]; then
    _info "config.d already exists"
    return 0            # ← skips everything else
fi
...
cat > "$AWSCONFD_CONFIG_DIR/00-defaults.conf" <<'EOF'   # ← unconditional
```

**Verified:** delete `scheme.conf` and re-run `init` — it is not recreated. The unconditional `cat >` is masked only by the early return; remove the early return without fixing it and `init` destroys a populated `00-defaults.conf`.

**Fix:** `init` performs each step independently, each guarded by "does this already exist?":

1. `mkdir -p` the three directories; `chmod 0700` each.
2. Back up + import `~/.aws/config` **only if** it exists and lacks the banner sentinel.
3. Create `00-defaults.conf` **only if absent**.
4. Create `scheme.conf` **only if absent**.
5. `chmod 0600` fragments it created.
6. Call `cmd_build`.
7. Print the "what next" block.

`init` currently does not build (spec §10.1 step 6). Add it.

### A3.3 `init` imports a generated file

If `~/.aws/config` already carries the banner, `init` copies it — banner and provenance comments included — into `99-imported.conf`, which then gets concatenated into the next build. Guard on the sentinel.

## A4. `build` performs no validation

Spec §6.2 step 3 is absent. `cmd_build` goes discover → assemble with nothing in between.

**Fix:** insert `_validate_blocking || return 3` between discovery and assembly. On failure, print every violation and **do not touch the output file** (test this explicitly).

Rules B1–B10 are currently either stubs returning 0 or, in B2's case, structurally incapable of returning nonzero. All of them need implementing per spec §9.2, and each must name the offending file and line.

Implementing B5, B7, B8 and W3 requires **top-level key extraction**, which does not exist. Add to the parser:

```
_parse_keys <file>   →  section_type|section_name|key|value|line_no
```

A line counts as a top-level key only if it has **zero leading whitespace** (spec §7 — indented lines are children of the preceding key, as in `s3 =` blocks). Strip trailing inline comments for validation only; never rewrite the fragment.

## A5. Build output defects

### A5.1 The trailing-newline check is dead code

```bash
if [[ -n $(tail -c 1 "$f" | od -An -tx1) ]]; then echo "" >> "$tmpfile"; fi
```

`od` emits a hex byte for *any* final byte, including `\n`, so the condition is always true. Result: an unconditional blank line after every fragment (harmless, but the guard does nothing).

The correct test relies on command substitution stripping trailing newlines:

```bash
if [[ -n "$(tail -c 1 "$f")" ]]; then printf '\n' >> "$tmpfile"; fi
```

Then emit the inter-fragment blank line separately and deliberately.

### A5.2 No `fragments-digest`

Spec §6.3. Compute sha256 over the concatenated fragment **bodies** (excluding the banner) using `sha256sum` or `shasum -a 256`; omit the line entirely if neither exists. Do not substitute anything non-deterministic.

### A5.3 Banner should carry a machine-readable sentinel

Add a stable line the tool can grep for unambiguously, e.g.:

```
# awsconfd-generated: v1
```

Everything in A3 keys off this. Human-readable banner text may then change freely.

### A5.4 Banner embeds an absolute `$HOME` path

Collapse `$HOME` to `~` (spec §6.3) so the built file is identical across machines with different usernames.

### A5.5 `--dry-run` prints a line count

Spec §10.4: print the assembled file to stdout, write nothing.

## A6. Argument parsing

### A6.1 Global flags are rejected after the subcommand

**Verified:** `awsconfd build --quiet` → exit 2. This is precisely the invocation the systemd unit uses.

Spec §10: globals accepted before *or* after. Implement a two-pass parse: sweep `$@` for known globals, removing them; the residue is `command [command-opts]`. Handle `--` as an end-of-options marker.

### A6.2 `--config-file` does not re-derive `--config-dir`

`AWSCONFD_CONFIG_DIR` is computed from `AWSCONFD_CONFIG_FILE` at line 13, before flags are parsed. Passing `--config-file /tmp/x/config` leaves the dir pointing at `~/.aws/config.d`.

Re-derive after parsing: if `--config-dir` was not given explicitly, set it to `$(dirname "$AWSCONFD_CONFIG_FILE")/config.d`.

### A6.3 Dead code

`_parse_globals` and `_discover_fragments` are never called. `main` reimplements the first; `cmd_build` reimplements the second with `find … -type f | sed`. Delete the duplicates, keep one implementation of each, and route `cmd_build`, `cmd_list`, `cmd_status`, `cmd_doctor` through `_discover_fragments`.

`find … -type f` also **excludes symlinks**, which spec §5.2 requires be followed. `-type f` → `-type f -o -type l`, or just use the glob with `[[ -f $f ]]` (which follows symlinks).

## A7. Parser gaps

- `;` comments are not skipped (spec §7 recognises `#` and `;`).
- Whitespace trimming `section="${section#[[:space:]]}"` removes at most one character. Use `section="${section#"${section%%[![:space:]]*}"}"` and the mirror for the tail.
- No key extraction (see A4).
- `_parse_sections` emits a full path in one caller and a bare basename in another. Standardise on the full path; derive the basename at print time.

# PART B — Phase 2 Specification

Do not begin until Part A's tests pass.

## B1. The section-update primitive

Everything in Phase 2 rests on one operation: **replace exactly one `[section]` block inside a fragment, leaving every other byte of the file untouched.** Build this first, test it in isolation, then build the commands on top.

### B1.1 Section extent

Given a file and a target header (the raw text inside the brackets, e.g. `profile personal-admin`):

- `start` = the line number of the line matching `^[[:space:]]*\[<header>\][[:space:]]*$`.
- `next` = the lowest line number greater than `start` matching `^[[:space:]]*\[` .
- `end` = the last line in `[start, next-1]` (or `[start, EOF]` if there is no `next`) that is **neither blank nor a comment**.

The replaced region is `[start, end]`. Blank lines and comments trailing the section are preserved verbatim, because a comment immediately above the *next* header belongs to that next section, and the user wrote it there.

If `start` is not found, the section does not exist.

### B1.2 The three operations

```
_section_exists  <file> <header>              → 0/1
_section_extent  <file> <header>              → "start end"   (stdout)
_section_read    <file> <header>              → the block, stdout
_section_upsert  <file> <header> <body_file>  → rewrite in place
_section_delete  <file> <header>              → rewrite in place
```

`_section_upsert` assembles `head(1..start-1) + new_block + tail(end+1..EOF)` into a `mktemp` file beside the target, `chmod 0600`, `mv`. Guard `start == 1` (no head). Guard `end == EOF` (no tail). If the section does not exist, append: ensure the file ends with a newline, emit a blank line, then the block.

Do **not** use `sed -n '1,Np'` with an unguarded `N=0`. A `while read` loop with a counter is clearer and has no portability caveats.

### B1.3 Section rendering

A section is rendered deterministically so that repeated `add-*` runs produce minimal diffs:

```
[profile personal-admin]
sso_session = personal
sso_account_id = 111122223333
sso_role_name = AdministratorAccess
region = eu-west-2
```

- Header, then keys, one per line, `key = value`, single spaces.
- Keys emitted in a **canonical order** per section type (below), then any keys not in the canonical list, in the order they were read from the existing section (so hand-added keys survive an update).
- No trailing blank line inside the block.

Canonical orders:

| Type | Order |
| --- | --- |
| `sso-session` | `sso_start_url`, `sso_region`, `sso_registration_scopes` |
| `profile` (sso) | `sso_session`, `sso_account_id`, `sso_role_name`, `region`, `output` |
| `profile` (assume-role) | `role_arn`, `source_profile`, `credential_source`, `mfa_serial`, `external_id`, `duration_seconds`, `region`, `output` |
| `profile` (static) | `region`, `output` |
| `profile` (process) | `credential_process`, `region`, `output` |

Nested/indented settings (`s3 =` blocks) inside an existing section are opaque: carry them through the update as trailing lines attached to their parent key.

### B1.4 Update-or-abort

When a target section already exists, `add-sso` / `add-profile` must:

1. Print the existing block.
2. Print the proposed block.
3. Prompt `Update this section? [y/N]` on `/dev/tty`.
4. Under `--yes`, proceed. Under `--non-interactive` without `--yes`, exit 2.

Never silently overwrite.

### B1.5 Tests

- Upsert the middle section of a three-section file; sections 1 and 3 are byte-identical afterwards.
- Upsert the first section; upsert the last section; upsert into a one-section file.
- Upsert a section followed by a comment block belonging to the next section — the comment stays with the next section.
- Upsert a section containing an indented `s3 =` block — the block survives.
- Upsert a section that does not exist — appended, exactly one blank line before it.
- Upsert a section into a file with no trailing newline.
- A section whose name contains a space (`[profile foo bar]`) — extent found correctly.
- Interrupting an upsert (`kill -INT`) leaves the original file untouched and no temp file behind.

## B2. The scheme

### B2.1 Format

`${XDG_CONFIG_HOME:-$HOME/.config}/awsconfd/scheme.conf`. Written once by `init`, never rewritten.

```ini
[awsconfd]
version = 1
strict  = false

[scheme]
00    = defaults
10    = sso-sessions
2x    = personal
30-39 = customer-a
55    = one-off-audit
```

Range key forms: `NN` (exact), `Nx` (decade `N0`–`N9`), `NN-MM` (inclusive, `NN <= MM`). Anything else → error naming the line.

### B2.2 Functions

```
_scheme_load                     → populates parallel arrays SCHEME_LO[] SCHEME_HI[] SCHEME_LABEL[]
_scheme_label_for_prefix <NN>    → label, or "" if uncovered
_scheme_range_for_label <label>  → "lo hi", or "" if undeclared
_scheme_next_free <lo> <hi>      → lowest unused two-digit prefix in range, or "" if full
_scheme_check_overlaps           → B9; exit 3 naming both entries
```

Bash 3.2 has no associative arrays. Use three parallel indexed arrays and a linear scan. There will be at most a few dozen entries; do not optimise this.

Prefix extraction from a filename: the leading two characters, only if both are digits.

### B2.3 Enforcement

- No `scheme.conf` → no scheme declared. Every prefix acceptable. No warnings. `--strict` in this state is a **usage error** (exit 2).
- `strict = false` (default) → a fragment whose prefix is uncovered emits **W2**, naming the nearest declared range. Build proceeds.
- `strict = true` or `--strict` → the same condition is **B10**. `build` exits 3; `add-profile` exits 2 and prints the next free prefix in the correct range.
- Range full during allocation → error naming the range, suggesting the manifest be widened.

## B3. `add-sso [<name>]`

```
awsconfd add-sso [NAME]
    --start-url URL
    --sso-region REGION
    --scopes SCOPES          (default: sso:account:access)
    --file FRAGMENT          (default: the fragment for the `sso-sessions` label,
                              else 10-sso.conf)
    --non-interactive
    -y, --yes
```

Flow:

1. Resolve the target fragment. If `--file` absent, look up the `sso-sessions` label in the scheme; if the range holds exactly one existing fragment, use it; if it holds none, create `<lo>-sso.conf`; if it holds several, prompt (or error under `--non-interactive`).
2. Gather values: flags win, then prompts on `/dev/tty`, then defaults.
3. Validate: `sso_start_url` must be `https://`-prefixed (warn if it does not end in `/start`); `sso_region` must be non-empty and match `^[a-z][a-z0-9-]+$`.
4. Render the block (B1.3). If the section exists, update-or-abort (B1.4).
5. `_section_upsert`.
6. `cmd_build`.

Prompts, in order: session name → start URL → SSO region → registration scopes.

## B4. `add-profile [<name>]`

```
awsconfd add-profile [NAME]
    --type sso|assume-role|static|process     (default: sso)
    --file FRAGMENT | --label LABEL
    --non-interactive
    -y, --yes

  sso:          --sso-session NAME  --sso-account-id ID  --sso-role-name ROLE
  assume-role:  --role-arn ARN  --source-profile NAME | --credential-source SRC
                --mfa-serial ARN  --external-id ID  --duration-seconds N
  static:       (no credential flags — see below)
  process:      --credential-process CMD
  all:          --region REGION  --output FORMAT
```

Flow:

1. Determine type. Interactively, present the four options with `sso` first and default.
2. **sso** — enumerate defined `[sso-session]` blocks. If none, offer to run `add-sso` now (or, under `--non-interactive`, exit 3 with **B4**). Validate `sso_account_id` matches `^[0-9]{12}$`. `sso_role_name` non-empty.
3. **assume-role** — `role_arn` matches `^arn:aws[a-z-]*:iam::[0-9]{12}:role/`. Exactly one of `source_profile` / `credential_source`. `credential_source` ∈ {`Environment`, `Ec2InstanceMetadata`, `EcsContainer`}. If `source_profile` is given, it must name a defined profile (or `default`) — else **B8**.
4. **static** — prompt for the *name of a profile in `~/.aws/credentials`*, and write only `region` and `output` into the fragment. Print an explanatory note that credentials live in `~/.aws/credentials`, which `awsconfd` does not manage. **Never prompt for an access key or secret key. Never open `~/.aws/credentials`.**
5. **process** — `credential_process` command string. Warn that it may embed a secret and that the fragment is mode `0600` but is not encrypted.
6. Resolve placement: `--file` wins; else `--label` → `_scheme_range_for_label` → `_scheme_next_free`; else prompt with the lowest free prefix overall as the default.
7. Render, update-or-abort, upsert, build.

## B5. `apply --spec <file>`

```
awsconfd apply --spec FILE|-    [--force] [--dry-run] [--strict]
```

1. If `-`, slurp stdin to a `mktemp` file (multiple passes are needed).
2. Parse three control sections: `[awsconfd]` (`version`, `strict`), `[awsconfd:scheme]`, `[awsconfd:layout]`. Reject `version` != 1.
3. `[awsconfd:scheme]` → write `scheme.conf` **only if absent**. If present and different, print the differences, keep the existing file, and continue (unless `--force`, which overwrites after confirmation).
4. `[awsconfd:layout]` keys are **raw section headers** (`default`, `profile foo`, `sso-session bar`); values are fragment filenames. Multiple sections may map to the same fragment.
5. Every remaining section is a payload section. For each, in file order:
   - Target = layout entry, else `_scheme_next_free` for the label inferred from the section type (`sso-session` → the `sso-sessions` label; `profile` → prompt or, non-interactively, W2 + lowest free prefix). Warn whenever placement is inferred.
   - If the section already exists **in any fragment**: skip and report `SKIP [profile foo] (already in 20-x.conf)` unless `--force`, in which case upsert into its **existing** fragment, not the layout target. Do not move sections between fragments; that is a manual operation.
   - Else `_section_upsert` into the target, creating it `0600` if new.
6. Run the blocking validators. Then `cmd_build`.
7. `--dry-run` prints the full plan — `CREATE 20-personal-admin.conf [profile personal-admin]` / `SKIP …` / `UPDATE …` — and writes nothing, including `scheme.conf`.

An `apply` that fails validation must leave `config.d` in its original state. Simplest correct approach: copy `config.d` to a `mktemp -d` staging directory, apply there, validate there, and only then `mv` the changed fragments into place. Do this; partial application is worse than no application.

## B6. `watch`

```
awsconfd watch --install [--with-timer] [--interval SPEC]
awsconfd watch --uninstall
awsconfd watch --status
```

### B6.1 Self-path resolution

The unit's `ExecStart` must be an absolute path. `readlink -f` and `realpath` are forbidden.

```bash
_self_path() {
    local p="$0"
    case "$p" in
        */*) ;;                             # has a slash: use as-is
        *)   p="$(command -v "$0")" ;;      # found via PATH
    esac
    printf '%s/%s\n' "$(cd "$(dirname "$p")" && pwd -P)" "$(basename "$p")"
}
```

Verify the result is executable before writing it into a unit. Refuse to install otherwise.

Also resolve `sleep` once: `command -v sleep` → absolute path for `ExecStartPre`. `/bin/sleep` is not universal.

### B6.2 Init-system detection

```
Darwin + launchctl present            → launchd
systemctl present
  AND (systemctl --user is-system-running returns anything but a failure to
       connect, OR $XDG_RUNTIME_DIR/systemd/private exists)
                                      → systemd user
otherwise                             → neither
```

"Neither" is **not an error**. Exit 0, install nothing, and print the Layer 3 recommendation (`eval "$(awsconfd hook bash)"`) plus a note that `awsconfd build` always works manually. WSL without systemd, containers, and minimal servers land here.

### B6.3 systemd

Written to `${XDG_CONFIG_HOME:-$HOME/.config}/systemd/user/`.

`awsconfd-build.service`:

```ini
[Unit]
Description=Rebuild ~/.aws/config from ~/.aws/config.d
Documentation=https://github.com/<owner>/awsconfd

[Service]
Type=oneshot
ExecStartPre=<abs path to sleep> 0.5
ExecStart=<abs path to awsconfd> --quiet build
Nice=10

[Install]
WantedBy=default.target
```

Note `--quiet build`, not `build --quiet`, until A6.1 lands. After A6.1 either works; prefer `build --quiet` for readability and add a test asserting it exits 0.

`awsconfd-build.path`:

```ini
[Unit]
Description=Watch ~/.aws/config.d for changes
Documentation=https://github.com/<owner>/awsconfd

[Path]
PathChanged=<abs path to config.d>
Unit=awsconfd-build.service

[Install]
WantedBy=default.target
```

`PathChanged` takes an absolute path; `%h` only helps when the directory is genuinely under `$HOME`. Substitute the resolved `AWSCONFD_CONFIG_DIR`.

Install:

```bash
systemctl --user daemon-reload
systemctl --user enable --now awsconfd-build.path awsconfd-build.service
```

Enabling **both** is deliberate: the `.path` unit is Layer 1, the `.service`'s own `WantedBy=default.target` is Layer 2 (a build at session start). Layer 2 is what repairs every state the watcher structurally cannot observe.

**Idempotency:** compare the generated unit text against the installed file; if identical, skip the write and skip `daemon-reload`. Re-running `watch --install` must be silent and cheap.

`--with-timer` additionally writes `awsconfd-build.timer` (`OnCalendar=hourly`, `Persistent=true`, default `--interval hourly`) and enables it, and prints the `loginctl enable-linger $USER` advice for headless hosts.

### B6.4 launchd

`~/Library/LaunchAgents/com.<owner>.awsconfd.plist`, with `ProgramArguments` = the absolute path + `build` + `--quiet`, `WatchPaths` = the absolute `config.d`, and `RunAtLoad` = true (Layer 2).

Load:

```bash
launchctl bootstrap "gui/$(id -u)" "$plist" 2>/dev/null \
  || launchctl load -w "$plist"
```

Unload mirrors this with `bootout` / `unload -w`.

`--with-timer` sets `StartInterval` (seconds; `--interval 3600`).

**Print, on macOS, at install time:** `WatchPaths` on a directory fires on directory-mtime changes — create, delete, rename. An in-place append (`echo >> file`) may not change the directory mtime and may not fire. Editors that save via write-temp-then-rename are unaffected. Recommend the shell hook.

### B6.5 `watch --status`

For each layer: installed? enabled? active? Is the `ExecStart` path still an existing executable? (Catches a moved or `self-update`d binary.) Report the `config.d` path baked into the unit against the currently resolved one and warn if they differ.

### B6.6 `watch --uninstall`

Disable and remove exactly what `--install` created; `daemon-reload`. Never remove a unit the tool did not write — check for a sentinel comment line in the unit file before deleting.

### B6.7 Tests

- No `systemctl`, no `launchctl` → exit 0, nothing written, Layer 3 message present in output.
- Generated units contain an absolute, existing, executable `ExecStart`.
- Second `--install` writes nothing and does not call `daemon-reload`.
- `--uninstall` leaves the unit directory as it was found.
- A unit file lacking the sentinel is not deleted.

Unit *installation* cannot be tested in CI without a session bus. Test the generated **text** against a golden file, and put the actual `systemctl` calls behind a single `_systemctl()` wrapper that the test suite can stub.

## B7. `hook` — rewrite

The delivered hook shells out to `awsconfd status --check` at every shell start and *warns*. Neither is right: Layer 3's job is to **rebuild**, cheaply, at every prompt.

Emit shell-specific code that does the staleness check with builtins only — `[[ a -nt b ]]` forks nothing:

```bash
# hook bash
_awsconfd_stale() {
    local d="${AWSCONFD_CONFIG_DIR:-$HOME/.aws/config.d}"
    local c="${AWS_CONFIG_FILE:-$HOME/.aws/config}"
    [[ -n ${AWSCONFD_HOOK_DISABLE:-} ]] && return 1
    [[ -d $d ]] || return 1
    [[ -f $c ]] || return 0
    [[ $d -nt $c ]] && return 0
    local f
    for f in "$d"/*.conf; do
        [[ -e $f ]] || continue          # unmatched glob
        [[ $f -nt $c ]] && return 0
    done
    return 1
}
_awsconfd_hook() { _awsconfd_stale && awsconfd build --quiet; return 0; }
case "${PROMPT_COMMAND:-}" in
    *_awsconfd_hook*) ;;
    "")  PROMPT_COMMAND="_awsconfd_hook" ;;
    *)   PROMPT_COMMAND="${PROMPT_COMMAND%;};_awsconfd_hook" ;;
esac
```

zsh: identical `_awsconfd_stale` (zsh's `[[` supports `-nt`), registered via

```zsh
autoload -Uz add-zsh-hook 2>/dev/null && add-zsh-hook precmd _awsconfd_hook \
  || precmd_functions+=(_awsconfd_hook)
```

In the common case (nothing stale) this forks zero processes. `awsconfd` is invoked only when a rebuild is actually needed.

`_awsconfd_hook` must always `return 0` — a non-zero `PROMPT_COMMAND` corrupts `$?` for the user's prompt.

Test: source the emitted bash hook in a subshell, assert `PROMPT_COMMAND` is appended rather than clobbered, assert sourcing twice does not double-register, assert `_awsconfd_stale` returns 1 on a fresh build and 0 after `touch`.
